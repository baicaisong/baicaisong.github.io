用hexo和GitHub Pages 来建立自己的博客

Posted on 2019-03-05 Edited on 2021-02-21

环境

  • git
  • node.js
  • hexo
    mkdir hexo 新建文件夹
    cd hexo
    yarn global add hexo 或 npm i -g hexo 全局安装 hexo

创建本地文件

  1. hexo init 初始化一系列文件
  2. hexo g 生成静态页面
  3. hexo s 启动本地服务
    在网页输入地址 localhost:4000 就可以看到本地的网页啦
  4. hexo new "new-page" 创建一篇文章
  5. <!--more-->标签可以使文章在主页中只展示前半部分

使用主题

  1. git clone https://github.com/theme-next/hexo-theme-next themes/next 下载主题
  2. 修改项目配置文件 _config.yml 中的 theme: next 选择此主题
  3. 修改主题配置文件 theme/next/_config.yml 中的 scheme: Mist 选择喜欢的外观
  4. 代码块样式美化：
    想要 Mac Panel 风格的代码块样式，NexT 自带了。
    修改主题配置文件 theme/next/_config.yml 中的 codeblock.copy_button.style: mac

GitHub Pages

  1. 新建一个仓库,仓库名必须为 [yourname].github.io

  2. 新增 SSH 密钥到 GitHub 帐户: 教程

  3. 修改项目配置文件 _config.yml 中的内容

  1. 上传到 github
    安装发布插件:yarn add hexo-deployer-git或npm install hexo-deployer-git --save
    执行提交：
  1. 修改package.json文件，scripts 下加一行命令：
    "d": "hexo clean && hexo generate && hexo deploy"
    每次执行yarn d，就等于执行第四步中提交命令。

使用 GitHub Actions 实现自动部署

为原代码新建一个仓库：

仓库[yourname].github.io.git中只保存编译后的文件，所以需要再建一个仓库来保存原 markdown 文件.
新建一个仓库名为 blog ,把源代码 push 到此仓库

配置密钥

将之前关联到 github 的 ssh 密钥分别存入两个仓库的 secret 中

  1. /blog/settings/secrets 下 New secret
  • Name :HEXO_DEPLOY_PRI
  • Vaule : cat ~/.ssh/id_ed25519里的数据
  1. /yourname.github.io/settings/secrets下 New secret
  • Name :HEXO_DEPLOY_PUB
  • Vaule : cat ~/.ssh/id_ed25519.pub里的数据

编写 action 脚本

  1. 在 blog 仓库根目录下创建 .github/workflows/deploy.yml 文件，内容如下:

  1. 参数说明

    • name: 名字
    • on: 触发条件，此处指当 push master 分支时会执行
    • env: 环境变量，会在 jobs 中调用
    • jobs:任务列表。
      • name 任务名称
      • runs-on 任务所需容器，可选值：ubuntu-latest、windows-latest、macos-latest。
      • strategy
      • steps 任务步骤

  2. steps 说明

    • name:步骤名
    • uses:所要调用的第三方 Action，查看更多。
      此处用了两个 Action：actions/checkout@v2（实现 Checkout 一个 git 仓库到容器。） & actions/setup-node@v1（可以配置容器 Node.js 环境）
    • with:传给 Action 的参数，具体可以查看所使用 Action 的说明。

  3. steps 详解

不带 with 参数时默认操作当前仓库，此时将此仓库 checkout 到容器中

因为此仓库中的 theme 是 git clone 下来的，所以还要复原 theme/next 中的代码。
此命令的意思为将第三方仓库 git@github.com:theme-next/hexo-theme-next.git 的 master 分支 checkout 到容器 themes/next 下。

安装 Node.js 版本 v14 到容器中

接着要还原本地的 configuration，在~/.ssh下的两个文件：

  • id_ed25519:私钥，同配置在 repo blog 的 secrets
  • known_hosts

因为 theme/next 中的 config 是默认的，但是我们有改过配置。所以提前将 theme/next/_config.yml 复制到根目录下重命名为_config.theme.yml，此时此容器会更新主题配置文件

最后安装依赖并发布

使用个人域名

  1. 先选择一个自己喜欢的域名购买（购买后需要实名认证）

  2. 认证成功后需要解析域名

  3. 点击解析

  4. 点击添加纪录

  5. 记录类型选择 CNAME
    记录值填写 github 的地址

  6. 在本地博客文件夹中的 source 文件中新建一个 CNAME 文件，在里面写上你的域名,然后再部署一次 yarn d

  7. 进入 github 项目，点击 settings，往下拉找到 github pages，在 custom domain 中填写你的域名并保存，成功后就可以通过你的域名进入个人网站了
